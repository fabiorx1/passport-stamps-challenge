AI-Powered Passport Stamp Analyzer üõÇ

This project automates the extraction and validation of information from passport stamps. It uses a composition of small, specialized AI models, backed by a Large Language Model (LLM) like Gemini 2.5 Flash. The LLM provides correct data when smaller models fail, creating a continuous learning loop to improve accuracy over time.

Solution Architecture ‚öôÔ∏è

The system is designed as a sequential pipeline, where each step processes the passport image to isolate and interpret the stamps.

1. Binary Classification
Decides if the image is a passport page.
2. Page Segmentation
Splits the main identification page from pages with stamps.
3. Stamp Localization
Finds the precise position and rotation angle of each stamp.
4. Optical Character Recognition (OCR)
Extracts the date and other text from the isolated stamp.
5. Validation & Re-training
Checks data consistency, standardizes date formats (e.g., to YYYY-MM-DD), and sends user feedback to a re-training pipeline.

Methodology & Core Idea üí°

Standard OCR models like Microsoft's TrOCR perform poorly on the varied fonts, angles, and ink patterns of passport stamps. My benchmark showed this clearly.

The core of this solution is using Gemini 2.5 Flash to bridge this gap. The LLM can accurately read the stamps, providing a high-quality "answer key." This data serves two purposes:

  • Fallback System: When a specialized model is uncertain, the query is passed to the LLM for a reliable answer.
  • Training Data Generator: The verified outputs from the LLM are used to continuously fine-tune the smaller, more efficient models.

This approach creates a self-improving system. Furthermore, because passport stamps naturally form visual clusters based on geography (e.g., Arabic, East Asian, or Latin American stamps share design elements), we can train even more specialized models for each cluster. A "router" model would first identify the stamp's likely region and then pass it to the corresponding expert model for highly accurate OCR.

Next Steps üöÄ

The immediate next step is to take the benchmark to its logical conclusion:

  1. Fine-tune Microsoft TrOCR: Train the `trocr-base-printed` model using the accurate OCR dataset generated by Gemini. This will create a powerful, specialized OCR model for passport stamps.
  2. Develop Vision Models: Implement the classification, segmentation, and localization models to build out the full pipeline.
  3. Build the Feedback Loop: Create the validation layer and the re-training pipeline to allow the system to learn from its mistakes.
  4. Develop Regional Models: Implement the "router" and specialized models for different stamp clusters to maximize accuracy.
